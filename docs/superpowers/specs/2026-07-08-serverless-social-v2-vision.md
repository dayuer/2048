# 无服务器·近场社交游戏 App（V2 愿景与分解）

日期：2026-07-08
仓库：https://github.com/dayuer/2048.git
关系：V1《离线时刻伴侣》的**进化**——保留「离线 / 隐私 / 无账号 / 无云」的灵魂作为差异化，把外壳从「断网时段容器」放宽为一个日常可用、像微信的近场社交游戏 app。

## 北极星

**一个无服务器、近场、隐私优先的游戏社交 app：**

- **随时能玩**：跟一个**本地 AI 对手**玩 2048 这类游戏，永远不用等人。
- **踩到人就热闹**：蓝牙 mesh 发现**身边**在用同一 app 的人。
- **连上就开战、能聊天**：跟发现的某个人**实时 1v1 对战** + **1:1 私聊**，全程设备到设备，无服务器、无账号。
- 飞机 / 断网是它最闪光的场景，但不再是唯一框架。「断网时段」从「整个 app」降为其中一个专注模式。

灵魂不变：诚实、克制、无在线化牵引；差异化来自「无服务器 / 近场 / 隐私」这条别人难抄的路径。

## 为什么这样收敛（brainstorm 结论）

- **AI = 本地游戏对手 bot**（expectimax 类算法），**不是 LLM**：纯本地、免费、省电，保住无服务器灵魂（V1 已排除端侧 SLM）。
- **对战 = 实时直连 1v1**：因为改为「发现近场某人 → 直连」，是**单跳**直连而非全舱多跳，实时可行；仅对战时启用高功耗无线电（MultipeerConnectivity / AWDL），有界、前台、可接受。
- **聊天 = 1:1 私聊**：与 1v1 直连天然配对，最像微信私聊，MVP 最聚焦。
- **飞轮**：本地 AI 对战（solo 留人）让用户随时打开 app；近场发现把「身边的人」变成真人对战与私聊。solo 留人与近场社交互为因果。

## 分解与建设顺序

V2 是多个独立子系统，拆成子项目、按「去风险」顺序建设，每个各自 spec → 计划 → 实现：

| 顺序 | 子项目 | 内容 | 风险/价值 |
|---|---|---|---|
| **A（先做）** | 本地 AI 同种子实时竞速 | on-device 2048 AI 对手 + 单机「你 vs AI」对战屏，自适应难度 | 无网络风险、立即可玩、复用确定性引擎、为对战 UI 打地基 |
| B | 近场发现 + 身份/连接底座 | BLE 发现身边用户 + ephemeral 身份 + MultipeerConnectivity 握手 | 最高技术风险，先验证「能不能连上」 |
| C | 实时 1v1 对战 | 同种子实时竞速的真人版，跑在 B 的连接上、复用 A 的对战 UI | 建立在 A、B 之上 |
| D | 1:1 私聊 | 跑在 B 的会话上的消息 | 建立在 B 之上 |
| Shell | 微信式 IA | 游戏 / 附近 / 我 三 tab 容器，承载以上 | 承载层 |

**本轮先为子项目 A 写详细 spec**（见 `2026-07-08-local-ai-versus-design.md`）。

## 待后续处理（非本轮范围）

- **变现**：Journey Pass 的「Session 模式买断」在这个更社交的 app 里需重新定位——留作独立线程，暂不动。
- **舱内擂台（异步排行榜）**：先前探索过的「全舱同种子 + gossip 排行榜」被实时 1v1 + 私聊取代出 MVP；作为**可能的后期 ambient 层**保留在想法库，不进 V2 核心。
- **更多游戏类型**（「2048 这类」）：A 先做透 2048；同框架后可扩展其它确定性玩法。
- **Android / 跨平台**、聊天室群聊、走子日志反作弊：更后期。
