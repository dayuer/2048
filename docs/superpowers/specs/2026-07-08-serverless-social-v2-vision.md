# 无服务器·近场社交游戏 App（V2 愿景与分解）

日期：2026-07-08（同日修订：WhatsApp 式 IA、去掉断网容器、游戏插件化）
仓库：https://github.com/dayuer/2048.git
关系：V1《离线时刻伴侣》的**进化**——保留「离线 / 隐私 / 无账号 / 无云」的灵魂作为差异化；**「断网时段容器」模式整体移除**，外壳重构为 WhatsApp 架构模式的近场社交游戏 app。

## 北极星

**一个无服务器、近场、隐私优先的游戏社交 app：**

- **随时能玩**：跟一个**本地 AI 对手**玩 2048 这类游戏，永远不用等人。
- **踩到人就热闹**：蓝牙 mesh 发现**身边**在用同一 app 的人。
- **连上就开战、能聊天**：跟发现的某个人**实时 1v1 对战** + **1:1 私聊**，全程设备到设备，无服务器、无账号。
- 飞机 / 断网是它最闪光的场景之一，但不再有任何「断网仪式」框架。

灵魂不变：诚实、克制、无在线化牵引；差异化来自「无服务器 / 近场 / 隐私」这条别人难抄的路径。

## 核心架构决策（brainstorm 收敛）

- **IA = WhatsApp 架构模式**：**对话是脊柱**。四 tab：对话 / 附近 / 游戏 / 我。AI 对手是**置顶常驻对话**（如 WhatsApp 的 Meta AI）；真人对战与聊天都发生在 1:1 线程里，**对战是线程内的事件卡**（像发附件）。详见《WhatsApp 式外壳与 IA》spec。
- **去掉断网容器**：Session 外壳（setup→active→landed 仪式）、离线轻提示、「断网时段」产品概念整体移除。`SessionActivity` 契约与 GridGame 基本法**不受影响**（本就解耦）。
- **游戏 = 独立插件模块**：全部长在 **GridGame 基本法**（`Sources/GridGame/`，见其 spec）上——同一引擎契约（`GridGameEngine`）+ 同一渲染/时间线词汇，支持多样式玩法。2048 是第一个插件，消消乐等后续接入。游戏 tab 即插件库。
- **AI = 本地游戏对手 bot**（expectimax 类算法），**不是 LLM**：纯本地、免费、省电。AI 线程只有对战，**无假聊天**。
- **对战 = 实时直连 1v1**：近场发现某人后单跳直连（MultipeerConnectivity / AWDL），实时可行；高功耗无线电仅对战时启用。
- **聊天 = 1:1 私聊**：与 1v1 直连天然配对，落在同一根线程里。
- **飞轮**：本地 AI 对战（solo 留人）让用户随时打开 app → 设备保持前台、可被发现 → 近场把「身边的人」变成真人对战与私聊。

## 分解与建设顺序

| 顺序 | 子项目 | 内容 | 状态 |
|---|---|---|---|
| 0 | GridGame 基本法 | 状态原语 + 结算时间线 + `SessionActivity`/`GridGameEngine` 契约；2048 引擎对齐 | **进行中**（见其 spec 与 alignment 计划） |
| **1a** | Shell/IA（WhatsApp 式） | 四 tab 外壳 + 对话线程 + 事件卡 + 插件注册表 + 拆除 Session 容器 | spec 已立（本轮） |
| **1b** | A：本地 AI 同种子实时竞速 | on-device 2048 AI 对手 + 对战屏（从 AI 线程进入），自适应难度 | spec 已立 |
| 2 | B：近场发现 + 身份/连接底座 | BLE 发现身边用户 + ephemeral 身份 + MultipeerConnectivity 握手 | 待 spec |
| 3 | C：实时 1v1 对战 | 同种子竞速真人版，跑在 B 的连接上、复用 A 的对战屏与线程事件卡 | 待 spec |
| 4 | D：1:1 私聊 | 跑在 B 的会话上的消息，落进同一线程 | 待 spec |

1a 与 1b 并列第一梯队（互不阻塞：1a 造壳与线程，1b 造对手与对战屏；AI 线程把二者接起来）。**实现均待「0」的 alignment 计划落地后再动工**，避免与在建引擎踩脚。
