# 游戏 tab「小游戏中心」App Store 式布局设计文档

日期：2026-07-08
上位：`2026-07-08-whatsapp-ui-replica-design.md`（外壳 WA 设计系统不变，本文档只改游戏 tab 的陈列层）。
决策（用户已确认）：**布局按微信小游戏中心截图复刻；「每个游戏是一个 AI agent」仅指陈列文案拟人化（agent 头像 + 一句话安利语），点击行为不变；只陈列注册表里的真游戏；两个分区都做，按属性分**。

## 一句话概念

**把游戏 tab 从单调列表行升级为「游戏安利站」大行列表 + 「回合小游戏」横滑封面卡的双分区小游戏中心；每个游戏以 AI agent 拟人化陈列（头像 + 安利语 + 标签），数据全部来自 `GameRegistry` 真实插件，点击仍直接进游戏。**

## 数据层（`Sources/Games/GamePlugin.swift`）

`GamePlugin` 新增三个陈列字段：

| 字段 | 类型 | 含义 | 2048 取值 |
|---|---|---|---|
| `tagline` | `String` | agent 拟人安利语（单行） | 「滑到 2048 算你赢，我在棋盘里等你」 |
| `tags` | `[String]` | 分类标签 | `["休闲", "益智"]` |
| `tint` | `Color` | 头像/封面底色 | 橙色系（呼应 2048 棋盘皮肤） |

- 注册表 `GameRegistry` 仍是唯一数据源，视图纯消费。
- **不加**榜单名次、「N 个朋友玩过」等字段——没有真实数据源，按诚实原则不造假角标。

## 视图层（重写 `Sources/UI/GameLibraryView.swift`，仅此一个文件）

- 页面底改为系统分组灰底（`systemGroupedBackground`，同「我」tab）；导航大标题仍为「游戏」。
- **分区一「游戏安利站」**：灰底上的分区标题（粗体）+ 白色圆角大卡片容器（`secondarySystemGroupedBackground`）；内部每游戏一行：
  - 56pt 圆形 agent 头像（`tint` 底 + 白色 SF Symbol）；
  - 名称 20pt semibold + `tagline` 灰色单行截断 + 底部灰色 `tags` 标签串；
  - 点击行为不变：全屏 cover 进单人游戏。
- **分区二「回合小游戏」**：`supportsVersus == true` 的游戏横滑封面卡：
  - 卡片约 110×150：`tint` 渐变底 + 大号 SF Symbol 封面 + 左下角小圆头像；
  - 卡下方：游戏名 + 「单人 · 对战」副题；
  - 点击同样直接进单人游戏；该区为空时整区隐藏（含标题）。
- 文字/分隔色沿用 `WA` token，无新主题文件。

## 测试与验收

- 纯视觉 + 描述符字段扩展，无逻辑变更；现有全量单测保持绿；不为颜色/布局写单测。
- 模拟器浅色 + 深色截屏，对照参考截图验收双分区布局。

## 不做的事

- 不加占位假游戏、假榜单/假社交角标。
- 不做游戏详情页中转，不改点击行为与导航结构。
- 不动 `GameRegistry` 之外的数据模型、引擎与其他 tab。
