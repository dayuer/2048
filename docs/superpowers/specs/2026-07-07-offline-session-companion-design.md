# 离线时刻伴侣（Offline Session Companion）V1 设计文档

日期：2026-07-07
仓库：https://github.com/dayuer/2048.git

## 一句话概念

**一个本地优先的「离线 Session（断网时段）」伴侣**：用户在登机前（信号尚在）买下它，登机后它变成一个安静、有始有终、无打扰的「高质量断网容器」——把 2048 作为第一个核心玩法。无广告、无账号、无云端牵引。卖的不是游戏，而是「被好好度过的断网时间」。

## 战略定位（brainstorm 结论）

经过逐层收敛，本产品的定位由以下几个已确认的决策锁定：

- **产品形态**：以 2048 为跳板的全新产品概念，而非单纯的 2048 clone。
- **主攻痛点**：「在线化」——强制联网、必须登录、数据采集、云依赖。产品方向是本地优先 / 离线 / 隐私。
- **用户策略（两者兼顾）**：用「被动离线」场景（飞行、通勤隧道、信号盲区）作为获客漏斗，把用户转化为「主动选择离线」的长期用户。
- **产品灵魂（关键取舍）**：**诚实的离线伴侣**。反剥削的价值观本身就是卖点。商旅人群是滩头市场，而非价值体系。因此**排除**激励视频、排除强制云端对抗——它们与本产品的灵魂冲突。
- **交互骨架**：**Session 主轴**，而非「启动器」。单纯启动器在商业上毫无防御力，会被淹没在应用商店聚合包里。把「断网时段」本身作为产品单元，卖「高质量的离线时间」，仪式（而非任一玩法）成为习惯载体，让低频飞行者也能形成「这是我的断网 App」的身份认同。

## 目标用户与楔子

- **滩头市场**：高频出差的商旅人士。可触达（机场 / 候机室 / 航司触点）、注意力高度集中（被动离线）、付费意愿真实，且恰好高度重合于「对 App 疲劳」的人群（主动离线）。他们在一次飞行中同时命中两种离线模式。
- **价值观**：诚实、克制、尊重时间与隐私。反剥削姿态即营销。

## 设计的第一性要求：审美即产品

**这是本产品成败的关键，不是 UI 的收尾工作。** Session 外壳必须彻底摒弃常规游戏大厅的繁冗：

- 极致留白，高对比度排版，向顶尖商业设计看齐（参考「安静、克制、昂贵」的观感，而非游戏化的花哨）。
- 只有当界面审美达到极简与克制的标准，用户才会在潜意识里认同这是一个「高价值的断网容器」，从而为付费转化建立信任基础。
- 无红点、无 badge、无 FOMO、无自动播放、无诱导。安静是第一交互原则。
- 后续在实现阶段应调用 `design` skill，确立一个「有承诺的美学」，而非通用默认风格。

## V1 范围（最小可行性 = 验证商业闭环）

V1 的**唯一任务**：验证「高频商旅人群是否愿意为『仪式感 + 时间容器』付费」。若在仅有一个核心玩法支撑下商业化路径仍能走通，说明核心逻辑立住。

**V1 = Session 外壳 + 2048（作为 Hero 玩法）+ Journey Pass（买断变现）。**

### 依赖：2048 本体

2048 目前处于半成品状态（`Sources/Engine` 有 `Position/Tile/Direction`，尚无 `GameEngine`、`Persistence`、`GameCenter`、`UI`）。**完成 2048 本体（依据其现有实施计划 `docs/superpowers/plans/2026-07-07-2048-swift-app.md`）是本 V1 的前置条件。** 本文档只覆盖在 2048 之上生长出的 Session 外壳、Journey Pass 与二者的集成。

## 组件设计

在 2048 现有四层架构（Engine / Persistence / GameCenter / UI）之上，新增两个模块并扩展 UI：

```
Sources/
├── Engine/          # 既有：2048 纯逻辑
├── Persistence/     # 既有：UserDefaults 存档（扩展：Session 存档、Pass 权益）
├── GameCenter/      # 既有：可选、自愿的排行榜
├── Session/         # 新增：Session 生命周期与状态（纯逻辑，可单测）
├── Monetization/    # 新增：Journey Pass（StoreKit 2，权益本地持久化）
└── UI/              # 扩展：Session 外壳视图 + 既有 2048 视图
```

### Session（纯逻辑，可单测）

Session 是产品的骨架。它是一个**有始有终**的断网时段容器。

- `Session`（`Codable`）：`id`、`startedAt`、`plannedDuration: Duration?`（可空——用户可以不设时长）、`state`（setup / active / landed / closed）、`activityLog`（本次 Session 内做过什么，仅本地）。
- `SessionController`（`@Observable`，编排层）：
  - `begin(duration:)`：进入 active，触发「安静环境」（见 UI）。
  - `pause()` / `resume()`：应对颠簸、供餐、广播打断——**进度绝不丢失**。
  - `land()`：进入 landed，展示克制的「你已落地」收尾。
  - Session 内的活动切换（V1 仅 2048，但接口为未来的程序化组件预留）。
- **离线检测策略**：以**手动**进入 Session 为主（用户按「开始断网时段」）。可*可选地*用系统网络状态（`NWPathMonitor`）做「检测到你离线了，要开始一个 Session 吗？」的轻提示，但绝不强依赖飞行检测（不可靠且易误触）。

### Monetization — Journey Pass（StoreKit 2）

**诚实变现的唯一杠杆。** 免费下载，游戏本体（2048）**永不被付费墙拦住**——付费买的是 **Session 模式**（仪式容器 / 安静环境 / Session 收尾与本地统计），而不是游戏本身。

- **购买时机**：候机室（设备仍在线）。这是本产品商业模式的关键——在注意力与购买意图的峰值、也是最后一个联网时刻完成变现。
- **权益结构（V1 的核心待验证假设）**：
  - **V1 采用一次性买断（永久解锁 Session 模式）**，而非按次计时的通行证。理由：重复的按次微收费带有制造紧迫感的意味，与「诚实」灵魂相悖；一次性买断（建议定价档位如 ¥18 / \$2.99–4.99）仍是候机室冲动消费，且干净、可信、无牵引。
  - 备注：用户最初设想的「\$0.99 解锁 8 小时」按程通行证作为**已记录的备选变现结构**，可在 V1 验证后作为 A/B 实验，不进入首个可交付版本。
- **实现**：StoreKit 2，非消耗型 IAP。购买在线完成，**权益本地持久化**（`Persistence`），此后完全离线可用。支持 `AppStore.sync()` 恢复购买。
- **无账号**：权益绑定 Apple ID 的 StoreKit 交易，不需要自建账号体系。

### Persistence（扩展既有模块）

- 既有：2048 局面存档、最高分、最大方块。
- 新增：当前 Session 存档（进度绝不丢失的落地保证）、Journey Pass 权益状态。

### GameCenter（既有，语义收敛为自愿）

- 保留 2048 的双排行榜，但在 Session 语境下**完全自愿、绝不作为门槛**：落地后可选「同步这次成绩」，永不强制、不弹窗骚扰、不做全服自动对抗。这与「不重新引入在线化」的原则一致。

### UI（扩展）

- **Session 外壳**（新增，审美即产品）：
  - 进入态：极简的「开始一个断网时段」入口（可选设时长）。
  - Active 态：安静环境——大面积留白、高对比排版、无红点无 badge，把 2048 承载其中。
  - 收尾态：克制的「你已落地」——本次 Session 做了什么（本地）、可选自愿同步。
- **2048 视图**（既有）：作为 Session 内的 Hero 活动嵌入。
- 输入 / 动画沿用 2048 既有设计。

## 数据流

1. 候机室（在线）：用户购买 Journey Pass → StoreKit 校验 → 权益写入本地 Persistence。
2. 登机（离线）：用户手动开始 Session（或响应可选的离线轻提示）→ `SessionController.begin`。
3. 飞行中（离线）：在安静环境里玩 2048；打断时 `pause/resume`，进度实时存档。
4. 落地（恢复在线）：`land` 收尾 → 可选自愿同步 Game Center 成绩。

## 错误处理与边界

- **购买失败 / 未联网就想买**：静默降级，提示「需要联网完成购买」，绝不打断游戏；2048 免费部分始终可玩。
- **权益校验**：离线时以本地持久化权益为准；联网时 `Transaction.currentEntitlements` 校验并恢复。
- **Session 中断**：任何时刻锁屏 / 切后台 / 来电，进度绝对不丢。
- **网络检测误报**：离线轻提示可被永久关闭，绝不骚扰。

## 测试

- **Session 逻辑单测**（Swift Testing）：状态机（setup→active→landed→closed）、pause/resume 不丢进度、Session `Codable` 往返。
- **Monetization 单测**：用 StoreKit 2 的 `.storekit` 测试配置，验证购买 → 权益持久化 → 离线可用 → 恢复购买。
- **2048 引擎**：沿用其现有测试计划。
- UI 通过模拟器构建 + 手动验证；审美通过 `design` skill 的产出与人工审阅把关。

## 不做的事（V1 YAGNI）

- **不做**激励视频、任何广告、任何 dark pattern。
- **不做**强制云端对抗 / 全服自动排名 / 强制账号。
- **不做**端侧小模型（SLM）推理——其算力消耗与电池焦虑同飞行场景的物理限制根本冲突，彻底剥离出 V1。
- **不做**除 2048 外的其它玩法 / 程序化生成组件——留待 V1 验证商业闭环后再逐步引入。
- **不做**按次计时通行证（作为已记录的备选变现结构，V1 后再实验）。

## V1 之后的路线图（非本 spec 范围）

1. 程序化生成的安静玩法组件（确定性生成 ≈ 零电耗），扩充 Session 内容。
2. 端侧小模型作为**后期、可选、有边界**的特性（绝非引擎），谨慎使用。
3. Journey Pass 变现结构 A/B（一次性买断 vs 按程通行证）。
4. 面向「主动选择离线」人群的留存与身份化设计。
