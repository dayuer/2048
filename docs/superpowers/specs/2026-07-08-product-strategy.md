# 产品策略文档 — 无服务器·近场社交游戏 App

日期：2026-07-08
仓库：https://github.com/dayuer/2048.git
性质：**顶层战略**，统摄并框定《V2 愿景与分解》及各子项目 spec。基于对标市场研究（FireChat / Bridgefy / bitchat / GamePigeon / Momo / Zenly / Spaceteam）收敛而成。

---

## 一、摘要（先读这一页）

**我们赌的是**：一个**隐私优先、永远有得玩**的休闲游戏伙伴——核心日常价值来自「随时能跟一个好玩的本地 AI 对手对战、也能跟朋友异步来一局」，而**无服务器的近场联机（蓝牙直连/mesh）是一个在特定高密度断网场景里才闪光的差异化加成，不是增长引擎**。

**关键战略转向**（相对此前设计）：**主次翻转。** 此前把「蓝牙 mesh 发现身边陌生人 → 联机」当作产品的中心与增长发动机。市场证据表明这是过去十年反复失败的那条路。因此：

- **日常驱动 = 本地 AI + 单机/异步对战**（有价值哪怕身边一个用户都没有）。
- **近场联机 = 高光加成**，只在「人密集且断网」的稀有时刻（飞机、教室、会场、地铁）登场。
- **增长 = 搭现有关系图谱**（分享到聊天、邀请链接），而非冷启一个标准 app。
- **变现从第一天就想清楚**（不重蹈 Zenly 覆辙）。

一句话：**别做「又一个蓝牙 mesh 社交 app」；做一个「永远有好对手陪你玩、偶尔还能跟身边真人擦出火花」的隐私向休闲游戏 app。**

---

## 二、市场判断（对标研究结论）

产品的四个组成部分，市场履历截然不同：

| 组成 | 市场证据 | 结论 |
|---|---|---|
| 休闲 1v1 游戏嵌在聊天里 | GamePigeon 半年登顶 iMessage 榜、十年长青；微信「跳一跳」数周破 1 亿 DAU | **强**——但赢在**搭了现有关系图谱 + 零摩擦分发**，不是自己建网 |
| 近场陌生人社交 | 陌陌 60M MAU、$3B IPO | **强**——但靠 **GPS+服务器（城市级）+ 约会级动机 + 高密度城市**，非蓝牙近场、非游戏动机 |
| 同处一地的本地联机 | Spaceteam 4.5★、Metascore 91 | **可爱但小众**——玩的是**已经在一起的朋友**，非陌生人发现 |
| 蓝牙 mesh 作为增长引擎 | FireChat（2018–19 关停）、Bridgefy（沦为抗议niche）、bitchat（2025 名人光环仍只在断网/抗议中 spike，日常留存失败，被迫加联网层） | **失败**——网络效应冷启 + 电量 + 移动断连，日常留存不成立 |

**三个致胜要素，我们原设计一个都不占：**
1. **搭现有图谱**（GamePigeon 借 iMessage）——我们是冷启标准 app。
2. **和已在一起的人玩**（Spaceteam）——我们要发现陌生人。
3. **约会级强动机 + GPS 密度**（陌陌）——我们的动机是「玩个休闲游戏」，比约会弱得多。

**最尖锐的一条证据**：bitchat 2025 年由 Twitter 创始人带最大流量推出，beta 瞬间满员，最终仍**只在马达加斯加、尼泊尔的断网抗议中单周数万下载地 spike，日常留存不成立**，并被迫加 Nostr 联网层救密度。**连 Jack Dorsey 都没能把「蓝牙 mesh 社交」做成日常习惯**——这是给「mesh 优先」路线最响的警钟。

**它们没否定的两点（我们的机会）：**
- **AI 对手 / 单机 / 异步玩法，身边没人时也有价值**——天然免疫所有 mesh 社交的冷启死结。
- **隐私 / 无服务器是当下有需求的定位**（bitchat 的热度证明兴趣真实）。但注意它是**加分项不是主因**：Briar / Jami / Berty 仅靠隐私都长不大；Zenly 坐拥 **40M MAU 仍因从不变现而被关停**——警示别搞「先全免费，以后再说」。

## 三、定位与差异化

- **真正的 painkiller（止痛药）= 永远在线的好对手 + 好玩的独处时间。** 手机游戏最大的痛点之一是「想来一局却没人陪 / 要联网要登录要等匹配」。我们给的是**零等待、零账号、零联网**的即时对局。
- **Vitamin（加分项）= 隐私 / 无服务器 / 近场真人。** 用来做差异化叙事与高光时刻，不用来扛留存。
- **一句定位**：*The game that's always ready to play — no server, no account, no waiting. And when someone's near, magic happens.*

差异化护城河来自「无服务器 / 近场 / 隐私」这条工程与价值观都难抄的路径——但**护城河不等于增长引擎**。护城河守住差异化，增长靠 AI/单机的即时价值 + 分发楔子。

---

## 四、目标用户与滩头

- **滩头一（日常盘）**：厌倦「联网、登录、匹配、氪金」的休闲玩家——想随手来一局有对抗感的小游戏。核心价值对他们**当天、独自**就成立。
- **滩头二（高光场景楔子）**：**密集且断网**的封闭场景人群——航班、通勤地铁、教室、会场、露营。这是近场联机唯一短暂成立的地方，是**最佳 demo 与获客故事**，但**不能作为留存盘**（一年飞几次的人留不住只在天上可用的功能）。
- 明确取舍：飞机场景是**楔子和病毒故事**，不是**日常驱动**。日常靠 AI/单机。

---

## 五、分发策略（增长的真问题）

冷启一个标准 app 去建蓝牙陌生人网络 = 死路（见对标）。增长必须**搭车现有关系图谱**：

1. **对局结果/邀请可分享到任意聊天**（微信/iMessage/WhatsApp）——「我 2044 分，来挑战同一种子」带链接，对方点开即玩。这是 GamePigeon 模式的可移植版：**把别人的社交图谱借来做分发**。
2. **异步对战即分发**：跟已有好友异步来一局（错峰、无需同时在线），每一局都是一次拉新触点。
3. **（评估）iMessage / 系统分享扩展**：把「同种子小游戏」做成可在聊天里发起的扩展，直接站在 iMessage 的图谱与零摩擦分发上。
4. 近场发现只在滩头二场景补上「惊喜感」，不承担获客 KPI。

---

## 六、商业模式（第一天就想，别学 Zenly）

Zenly 40M MAU 仍因无变现被关停——**留存≠生意**。候选（保持「诚实、无 dark pattern」灵魂）：

- **一次性买断解锁「+」**（GamePigeon+ 即 $4.99 卖皮肤/模式/去广告的成熟路径）：更多游戏插件、AI 难度风格、对战主题。
- **游戏插件内购**：基础 2048 免费，进阶玩法/新游戏插件按个买断。
- **绝不做**：激励视频、强制账号、数据变现、制造 FOMO 的按次收费（与灵魂冲突）。
- 原 Journey Pass「Session 模式买断」随断网容器移除而解绑；变现锚点迁移到「游戏内容 / +」。

## 七、路线图与优先级（据战略重排）

| 阶段 | 交付 | 战略作用 |
|---|---|---|
| 0 | GridGame 基本法 + 2048 引擎对齐 | 插件底座（进行中） |
| **1（日常盘·先做）** | 本地 AI 同种子实时竞速（子项目 A）+ WhatsApp 式外壳（1a） | **建立冷启即成立的日常价值**——这是全部战略的地基 |
| 2（分发） | 结果/邀请分享到外部聊天 + 异步对战 | **借现有图谱增长**，验证「聊天里的游戏」拉新 |
| 3（高光加成） | 近场发现 + 直连 1v1 对战（子项目 B、C） | 在密集断网场景制造「惊喜」，做差异化与病毒故事 |
| 4（社交深化） | 1:1 私聊（子项目 D）、更多游戏插件 | 加厚关系与内容库 |
| 5（变现） | 「+」买断 / 插件内购 | 把留存变成生意 |

**与此前的关键差异**：蓝牙近场从「阶段 1 的中心」降到「阶段 3 的加成」；新增「阶段 2 分发」为一等公民。

## 八、风险与验证（含 kill 标准）

按风险从高到低，每条配一个**便宜的验证实验**和**止损线**：

1. **冷启日常价值不足**（最致命）。*验证*：只做阶段 1，看单机/AI 对战的 D1/D7 留存与人均对局数。*止损*：若 AI 对战撑不起留存，后面 mesh 更救不了——回炉玩法或叫停。
2. **分发进不去现有图谱**。*验证*：阶段 2 灰度分享链接，测邀请转化/K 因子。*止损*：K 因子长期 <0.3 且无机增长 → 重估品类。
3. **近场密度永远不够**（对标最强负面信号）。*对策*：从设计上**不依赖它留存**，仅作加成；*验证*：在真实航班/会场做一次 mesh 联机 demo，测「同场同 app 两人」的相遇率与对局完成率。*认知*：即便这条弱，产品仍靠 1、2 成立。
4. **隐私叙事叫好不叫座**。*认知*：当加分项经营，不做核心 KPI。
5. **变现拖延**（Zenly 教训）。*对策*：阶段 5 前就跑一次小样本付费意愿测试。

**元原则**：**产品的成立性绝不押在最不确定的 mesh 上。** 每个阶段都要能独立产出价值：阶段 1 就是一个能玩、能留人的单机对战游戏。

---

## 九、成功指标（分层）

- **日常盘（阶段 1–2）**：D7 留存、人均日对局数、AI 对战完成率；分享邀请的 K 因子。
- **高光加成（阶段 3）**：密集场景下的近场相遇率、直连对局完成率、「哇时刻」定性反馈。
- **生意（阶段 5）**：付费转化率、ARPU；对标 GamePigeon+ 的买断转化基准。
- **灵魂守护（全程）**：零账号、零服务器、数据不出设备——作为不可妥协的定性约束与营销资产。

---

## 十、不做的事（战略级 YAGNI）

- **不把蓝牙 mesh 当增长引擎**；不为「全舱多跳」这种最难又最不留存的能力过度投入。
- **不做冷启的纯标准 app 增长**（不搭现有图谱就不指望病毒增长）。
- **不做**假 AI 聊天（LLM）、强制账号、云同步、数据变现、激励视频、FOMO 收费。
- **不把飞机/断网当留存盘**——它是楔子和 demo。
- **不拖变现**到「以后再说」。
